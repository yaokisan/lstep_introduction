LステップフォームLP 作成手順書

回答フォーム機能を活用したランディングページ作成のプロセスと技術TIPS

1

はじめに：今回のゴール

この資料は、Lステップの回答フォーム機能を活用して、訴求力の高いランディングページ（LP）を作成する手順と、その際に使用した技術的なポイントをまとめたものです。

完成形のイメージ

  • Lステップのフォームでありながら、ヘッダーに作り込んだLPデザインを画像として表示させる。
  • フォーム送信後は、LINE登録を促す専用のサンクスページに遷移させる。
2

Lステップフォームの基本設定

まず、Lステップの管理画面から回答フォームを作成し、必要な項目（業種、氏名、役割など）を通常通り追加します。ここでの設定は標準的なものです。

3

Step 2：HTMLを画像化

作成したHTMLコードをブラウザで表示させ、**GoFullPage**のようなブラウザ拡張機能や、Chromeの検証ツール（右クリック > 検証 > ノードをキャプチャ）を使い、表示されたデザインを高解像度の画像としてキャプチャし、PCに保存します。

Step 3：Cloudinaryで画像をホスティング

Lステップのメディア機能に画像をアップロードすると、自動的に圧縮がかかり解像度が低下してしまいます。これを避けるため、外部の画像ホスティングサービス**「Cloudinary」**を利用します。

  1. Cloudinaryに無料登録し、ログイン。
  2. `Media Library > Assets` から、Step 2で作成した画像をアップロード。
  3. アップロードした画像のURLをコピーします。
4

Lステップへのヘッダー画像設定とCSS調整

ここが技術的なポイントです。Lステップのフォームに、外部サービスの画像を美しく表示させるための設定を行います。

設定手順

  1. ヘッダー画像URLの設定: Lステップのフォーム編集画面で `カラー/デザイン設定 > ヘッダー画像URL` を選択し、Cloudinaryで取得したURLを貼り付けます。
  2. カスタムCSSの設定: `カラー/デザイン設定 > カスタムCSSを使用` を選択し、以下のCSSコードを貼り付けます。
/* ▼▼ ヘッダー画像をレスポンシブに表示させるためのCSS ▼▼ */

/* Lステップのデフォルトのタイトルと説明文を非表示にする */
.form-top .form-top-left {
    display: none;
}

.form-top .form-top-right {
    display: none;
}

/* ヘッダー画像の表示領域を調整 */
.form-top {
    background-position: center center;
    width: 100%;
    height: auto;
    /* 画像の縦横比を固定し、高さを自動調整させる */
    /* 計算式: [画像の幅] / [画像の高さ] */
    aspect-ratio: 1380 / 2112;
}

/* PC表示の際の高さを調整（任意） */
@media (min-width: 768px) {
    .form-top {
        /* PCでは高さを固定することも可能 */
        height: 300px; 
    }
}

最重要ポイント

`aspect-ratio: 1380 / 2112;` の部分です。これにより、スマートフォンの画面幅が変わっても、ヘッダー画像の縦横比が常に一定に保たれ、美しく表示されます。この数値は**「画像の実際の幅 / 画像の実際の高さ」**で算出するため、画像を変更した場合は、この数値を再計算して修正する必要があります。

5

サンクスページの設定

Lステップのフォームは、送信ボタン以外のカスタムボタンを設置できません。そのため、フォーム送信後のサンクスページを外部で作成し、LINE登録を促すフローを構築しました。

  • 作成ツール: Google AI Studio のビルド機能
  • 設定方法: Lステップのフォーム編集画面 `オプション > サンクスページURL` に、AI StudioでデプロイしたページのURLを設定します。
  • AI Studio 共有編集ページ: こちらからアクセス
  • 注意点: オンライン、東京、大阪でLINEの友だち追加リンクを分け、登録者をタグ付け管理したい場合は、それぞれ個別のサンクスページを作成・デプロイする必要があります。
!

補足：開催地ごとのヘッダー画像について

LP画像の上部にある「オンライン」「大阪」といった開催地のテキストは、現在**Canva**を使って各パターンの画像に直接文字入れをしています。将来的には、この部分もより効率的な運用方法を検討する必要があります。
